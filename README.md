**Project Title**

**Retail Customer Purchase Behavior Analysis using MySQL**

**Project Overview**

This project analyzes retail transaction data to understand customer purchasing behavior, product demand, and sales trends using MySQL. The analysis includes data cleaning, duplicate handling, customer segmentation, and advanced SQL queries to generate business insights.

The goal of this project is to demonstrate SQL data analysis skills commonly required in Data Analyst roles.

**Dataset**

The dataset consists of two tables:

**customer_details**

Column	Description
customer_id	Unique customer identifier,
sex	Gender of the customer,
customer_age	Age of the customer,
tenure	Number of years the customer has been associated

**basket_details**

Column	Description,
customer_id	Customer identifier,
product_id	Purchased product ID,
basket_date	Purchase date,
basket_count	Number of items purchased

**Project Workflow**
1. Data Exploration
Checked table structure,
Reviewed sample records,
Calculated total record counts,

2. Data Quality Checks
Identified:
Missing values,
Duplicate records,
Data inconsistencies

3. Data Cleaning
Removed duplicate transaction records.

4. Customer Purchase Analysis
Key analyses performed:
Total purchases by customers,
Most active customers,
Purchase frequency,
Product demand analysis,
Customer segmentation

5. Business Insights Generated
The analysis helps identify:
Top spending customers,
Most frequently purchased products,
Customer purchasing patterns,
Repeat customers,
Customer segmentation insights
